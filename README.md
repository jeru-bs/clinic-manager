# ניהול קליניקה

מערכת Web בעברית ובכיוון RTL לניהול קליניקה: מטופלים, יומן, תיעוד מפגשים, משימות, תשלומים וקבצים.

הגרסה הפעילה מתפרסמת דרך GitHub Pages מתוך תיקיית `docs`.

## מה קיים עכשיו

- ממשק עבודה בדפדפן עם סיידבר ימני.
- חיבור Google OAuth מתוך הדפדפן.
- בדיקת חשבון Google מורשה לפני טעינת נתונים.
- שמירת מטופלים, מפגשים, תשלומים, משימות וקבצים ב-Google Sheets וב-Google Drive.
- יצירת תיקייה נפרדת לכל מטופל בתוך תיקיית האחסון הראשית.
- העלאת קבצים לכרטיס מטופל ושמירתם בתיקיית המטופל בדרייב.
- מחיקת קבצים מכרטיס המטופל ומהדרייב לאחר אישור.
- תיעוד מפגש במסמך Google Docs ושמירת קישור בכרטיס המטופל.
- סנכרון מפגש שמור ל-Google Calendar.
- תצוגת יומן חודשית, כולל מפגשים קבועים שמחושבים מתוך הגדרת המטופל.
- שמירת מפגש קבוע מתוך היומן כמפגש רגיל עם סנכרון ותיעוד.
- חריגי יומן למפגשים קבועים: ביטול חד-פעמי, חופשות, חגים וימים חסומים.
- ניהול משימות ותשלומים.
- תזכורות בסיסיות מתוך משימות, עם תאריך תזכורת ותצוגה בדשבורד.
- יומן Audit מרכזי ו-Undo לפעולות שניתנות לביטול בטוח; פעולות חיצוניות בלתי-הפיכות מתועדות ללא כפתור ביטול מטעה.
- יצוא טבלאות, גיבוי מלא לקובץ או לדרייב, ושחזור מאומת עם גיבוי מקדים ו-rollback אוטומטי במקרה כשל.
- בדיקת תקינות מבנה Google Sheets ותיקון גיליונות/כותרות חסרים.

## טבלאות מערכת

המערכת משתמשת בגיליונות הבאים:

- `patients`
- `sessions`
- `payments`
- `tasks`
- `files`
- `schedule_exceptions`
- `audit_log`

מבנה העמודות מוגדר ב-`docs/app.js` בתחילת הקובץ.

## מה עדיין לא סגור

- רשימת חשבונות Google מורשים היא חובה; רשימה ריקה חוסמת גישה. אסימון הגישה נשמר לסשן בלבד, והמערכת מנסה לשחזר אוטומטית חיבור שכבר אושר במכשיר.
- הרשאות Google Sheets ו-Google Drive הן גבול האבטחה הקשיח של גרסת GitHub Pages. להגנה מלאה באמצעות סשן שרת נדרש להעביר את המוצר הפעיל לאירוח פרטי עם backend.
- בכל התחברות מורשית המערכת בודקת את גיליון הנתונים ותיקיות הקליניקה ומסירה אוטומטית הרשאות שיתוף ציבוריות מסוג `Anyone` לפני טעינת הנתונים.
- חריגי יומן קיימים, אך עדיין אין חישוב אוטומטי של לוח חגים ישראלי. חגים וחופשות מוזנים ידנית בהגדרות.
- תמלול AI להקלטות עדיין לא מחובר.
- חשבונית ירוקה עדיין לא מחוברת.

## הרצה מקומית

לבדיקת קוד סטטי:

```powershell
node --check docs\app.js
```

בדיקת שחרור מלאה:

```powershell
npm.cmd run release:check
```

הפקודה בודקת קוד סטטי, כללי אבטחת דפדפן, תהליכי Audit/Undo ותזכורות, תקלות Google, אימות גיבוי/rollback ו-lint.

אם עובדים על גרסת Next הישנה, יש להשתמש ב-`npm.cmd` ב-Windows:

```powershell
npm.cmd install
npm.cmd run dev
```

הקבצים המקומיים `work/` ו-`.env.local` אינם מיועדים לפרסום לגיטהאב.
